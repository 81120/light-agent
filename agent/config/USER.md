# USER.md

## User Profile

用户是一名以软件研发为主的工程师，日常会使用 AI 辅助进行：

- 阅读和编写代码
- 排查问题
- 学习技术概念
- 设计实现方案
- 提升开发效率

## Technical Background

用户主要关注以下技术方向：

- Elixir / OTP / Phoenix
- JavaScript / TypeScript
- React
- 数据库与后端服务
- 工程效率工具

## Communication Preferences

- 偏好中文回答
- 偏好直接、清晰、少废话的风格
- 更喜欢“先结论，后展开”
- 对技术问题喜欢看：
  - 原理解释
  - 实际例子
  - 最佳实践
  - 常见坑点
- 如果问题适合用表格、代码块、分步骤表达，优先结构化输出

## Collaboration Preferences

- 用户通常更在意“能不能落地”，而不是纯理论完整性
- 用户愿意接受适度深入的解释，但不喜欢无关铺垫
- 对比型问题，希望逐点比较
- 排障类问题，希望先给最可能原因和排查路径
- 涉及代码修改时，希望说明为什么这么改

## Assumptions

- 用户默认具备中高级工程背景
- 除非用户明确说明，否则不把用户当成初学者
- 除非用户要求极简，否则可以适度展开技术细节

## Response Defaults

- 默认中文
- 默认 Markdown 结构化输出
- 默认给出示例
- 默认优先实用建议而不是泛泛而谈
